# 订单处理脚本优化历史

---

## 2026-04-07

### 核心问题分析
发现了 pick 货位排序混乱问题：
- CC 货区后面紧跟着 HH 货区，从优化算法角度不应该发生
- single 类型中出现"2200 还没排完就塞了其他 zone"的情况

### 问题根源定位
1. **单个 UPC 组本身跨 zone 问题：
   - single 类型 284 个 group 中有 2 个（0.7%）包含多个 zone
   - multiple 类型 24 个 group 中有 4 个（16.7%）包含多个 zone
   - 这是仓库库存分布的原始数据问题

### `optimize-pick-batches.js` 重大改进

#### 1. 评分算法调整
- **modified `scoreCandidate` 函数**：
  - 从完全禁止不同货区前缀改为加大惩罚
  - sharedPrefixes 权重从 5000 增加到 10000
  - 新增无 sharedPrefixes 时的惩罚：如果没有共享货区前缀，扣减 10000 分
  - 这样在保证数量控制的前提下，尽量让同货区前缀的订单在一起

#### 2. single 类型优化策略重构
- **优化前问题**：使用启发式聚类算法，可能导致路径回退
- **优化后改进**：改用全局路径排序 + 箱容切分
- **新增 `compareSingleGroupsByPath` 函数**：
  - 优先按 dominantZone 排序
  - 再按 dominantShelf 排序
  - 最后按 anchorPosition 的 row/column/side 排序
- **重构 `optimizeSingleBatchesByPath` 函数**：
  - 撤销过于激进的货区前缀强制检查
  - 只按箱容分批，保持路径连续性
  - 确保从仓库头走到尾的最优路径

#### 3. multiple 类型启发式优化算法种子选择逻辑确认
- **种子选择策略**：优先选择库位靠前的 group
- **排序优先级**（`compareGroupsByAnchor` 函数）：
  1. dominantZone（zone 靠前
  2. anchorPosition.row（行靠前）
  3. anchorPosition.column（列靠前）
  4. anchorPosition.side（侧靠前）
  5. totalQty（件数多的优先）
  6. upcUnique（最后按 UPC 唯一标识）

### 两种算法对比
| 特性 | single（全局路径排序） | multiple（启发式聚类） |
|------|------------------------|------------------------|
| **适用场景** | group 之间独立，无绑定 | group 之间有组合绑定 |
| **排序方式** | 全局线性排序（类似蛇形路径） | 局部聚类（种子 + 最佳候选） |
| **路径连续性** | ✅ 最佳，从仓库头走到尾 | ⚠️ 一般，可能有回退 |
| **组合保持** | ❌ 不考虑 | ✅ 考虑 sharedShelves/sharedZones 等 |
| **实现复杂度** | 简单 | 复杂 |

### 新增辅助脚本
- `analyze-groups.js`：分析单个 group 是否包含多个 zone
- `verify-fix2.js`：验证修复效果的脚本

## 2026-03-29

### `optimize-pick-batches.js`
- 新增拣货批次优化脚本。
- 支持读取 `single` / `multiple` 两类数据结构。
- 保持 `upcUniqueList` 与 `orders` 的索引对应关系不变。
- 按每组最小 `upcUniqueList[i] + orders[i]` 作为重组单元。
- 优先按 `zoneName`、`shelfName` 进行聚合，尽量减少拣货员跨区移动。
- 保持每批最多 `15` 个 UPC 类别，可通过 `--box-size` 调整。
- `single` 类型下，组内订单按数量从大到小排序，例如 `*4` 在 `*1` 前。

### `process-ordersV4.js`
- 已集成 `optimize-pick-batches.js`。
- 执行 `node process-ordersV4.js xxx.json` 时，会先自动优化，再继续生成 PDF。
- 自动优化后的总 JSON 会直接覆盖原始输入文件。
- 新增 `--skip-optimize` 参数，用于跳过优化并保留原 JSON。
- 新增 `--box-size=15` 参数，用于控制优化阶段每批最大 UPC 数量。
- 处理前会输出优化统计信息，方便快速确认批次数、zone/shelf 切换次数。
- 修复 PDF 合成顺序：改为严格按 JSON 中订单遍历顺序合成，不再受文件名字典序影响。
- 下载的面单文件名新增组内序号前缀，进一步保证同组内顺序稳定。
- 延续 V4 已有的批次目录 + `u{upcIndex}` 文件名前缀策略，规避 Windows 大小写不敏感导致的重名覆盖问题。
